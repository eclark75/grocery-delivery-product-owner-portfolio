
# User Stories

## Grocery Delivery App User Stories

### User Story 1: User Registration
**As a** new customer,
**I want** to create an account,
**So that** I can save my information and place grocery orders.

**Acceptance Criteria**
- User can enter name, email, and password.
- User receives confirmation after successful registration.
- Duplicate email addresses are not allowed.

**Priority:** High  
**Story Points:** 5

---

### User Story 2: User Login
**As a** returning customer,
**I want** to log in,
**So that** I can access my account and previous orders.

**Acceptance Criteria**
- User can log in with a valid email and password.
- Invalid credentials display an error message.
- User remains logged in until they sign out.

**Priority:** High  
**Story Points:** 3

---

### User Story 3: Search for Products
**As a** customer,
**I want** to search for grocery items,
**So that** I can quickly find products I need.

**Acceptance Criteria**
- Search returns matching products.
- Search updates results as keywords are entered.
- If no products are found, an appropriate message is displayed.

**Priority:** High  
**Story Points:** 5

---

### User Story 4: Add Item to Cart
**As a** customer,
**I want** to add products to my shopping cart,
**So that** I can purchase multiple items in one order.

**Acceptance Criteria**
- Customer can add items to the cart.
- Cart displays product name, quantity, and price.
- Customer can update quantities or remove items.

**Priority:** High  
**Story Points:** 5

---

### User Story 5: Checkout
**As a** customer,
**I want** to complete my purchase,
**So that** I can receive my groceries.

**Acceptance Criteria**
- Customer enters delivery address.
- Customer selects a payment method.
- Order confirmation is displayed after successful payment.

**Priority:** High  
**Story Points:** 8
